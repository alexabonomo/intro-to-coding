What is a CMS?

What is a CMS?

A content management system (CMS) is software that helps users create, manage, and modify content on a website without the need for technical knowledge. In other words, a CMS lets you build a website without needing to write code from scratch (or even know how to code at all).

Why use a CMS?

  • So you don't have to code every time you update your site
  • When you are collaborating with others who don't know how to code

CMS Basics

Folder Structure

/content

The content of your website.

/site

Think of this folder as your project folder. It contains your templates, configuration, plugins and setup files (blueprints) for the Panel.

/kirby

The Kirby app. Great for studying the source code, but in general you should leave this folder untouched (apart from updating it when a new version is released).

/media

This folder is managed by Kirby and holds public images and thumbnails as well as plugin and Panel assets. Let Kirby do its work and leave this folder as it is. The media folder and the files in it are auto-generated.

you can create a separate folder called /assets to store media in

your content lives in your /content folder. kirby doesn’t need a database. every subfolder represents a page

The content for each page is stored in a text file together with additional resources like images, videos, or other documents. The  site.txt  in the content folder contains general site-wide information.

text files

templates to render your content

blueprints

you will use blueprints to configure layouts, create forms, set permissions, etc

Creating Content

Kirby is a flat-file CMS or a file-based content management system (CMS) that stores files and data in text files and folders rather than a database.

If you create a folder called blog as direct child of the content folder, it will be accessible via https://yourdomain.com/blog etc.

3 ways to upload content

  1. You can use the Panel to edit content and upload files.
  2. You can create files and folders manually in the file system and edit content files with the text editor of your choice.
  3. You can create and modify content programmatically using Kirby's API, for example, create pages from .csv files or from form content on the front-end.

Snippets with Slots

you can create ways to pass information into variables with slots

in your default.php, lets make a pop up window. add this

<div class="alert">
    <?= $slot ?>

    <button type="button">close</button>
    
</div>

and in your snippets folder, make a snippet called alert.php with this code

to define your own variable we can use the $slots variable

<button type="button"><?= $slots->button() ?></button>

and we can edit in default.php here

<?php snippet('alert', slots: true) ?>

    <?php slot('default') ?>
    <h2>hi there, welcome to my homepage</h2>
    <?php endslot() ?>

    <?php slot('button') ?>
    close
    <?php endslot() ?> 

<?php endsnippet() ?>
