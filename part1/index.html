Welcome!

As you take this course, you are going to dive deep into the make up of the web and code your own small little site and plant small little garden plots of internet space.

Once you learn the components of what makes up the internet, coding websites can start to feel like a handmade craft. Each line being written and composed carefully. This is called the handmade web.

Lets start out with a little bit of historical context.

Olia Lialina

Olia Lialina is a pioneer of net art born in Russia. Her network-based art includes My Boyfriend Came Back from the War (1996), Agatha Appears (1997), First Real Net Art Gallery (1998), and Last Real Net Art Museum (2000). She wrote A Vernacular Web in 2005 which talked about the old aesthetics of the World Wide Web I, the web during its amateur state.

According to Lialina, the amateur web felt boundless and hopeful. Many sites had big "under construction signs". It didn't mean that the site didn't work, it meant that someone was always there maintaining the site. Sites were playful and decorated with sparkly gifs and starry backgrounds. It was customized and expressive with cooperative effort to connect their sites to others.

In 2015, J. R. Carpenter wrote in his essay, The Handmade Web,

I evoke the term 'handmade web' to refer to web pages coded by hand rather than by software; web pages made and maintained by individuals rather than by businesses or corporations; web pages which are provisional, temporary, or one-of-a-kind; web pages which challenge conventions of reading, writing, design, ownership, privacy, security, or identity.

I evoke the term 'handmade web' in order to make a correlation between handmade web pages and handmade print materials, such as zines, pamphlets, and artists books.

Laurel Schwulstt

Laurel Schwulst is an artist whose focus is on the poetics of the internet. She wrote My website is a shifting house next to a river of knowledge. What could yours be?, which compares HTML to building the scaffolding of a house. She goes on to talk about a website as a metaphor – website as a room, as a shelf, as a plant, as a garden, etc.

Setting up Visual Studio Code

To begin handcoding HTML, we will be using an editor called Visual Studio Code.

Once you download that, you should open the editor and navigate to the extensions icon on the very left panel. It should be the last icon. Install these extensions:

  • HTML Boilerplate — A template for html
  • htmltagwrap — Highlight code and wrap elements around them quickly
  • Beautify – Indents code for legibility
  • Live Server – Every time you save, browser where you view page updates
  • Code Spell Checker – Good for copyediting.

Introduction to the
Command Line Tool

Throughout this course, you will get well acquainted with the command line tool.

For Mac users, we’ll use Iterm or the default Mac Terminal. Go to your search icon (or Cmd + Space) at the tool bar and search for "terminal". For Windows, download git bash. You can copy and paste a line in Git bash with Ctrl + Shift + C / V.

The program you use on Windows to access the command line is called “Command Prompt.” You can access this program from the Start menu, under “All Programs > Acessories > Command Prompt.” The command prompt looks different on Windows from how it looks on UNIX-ish machines; for one, the prompt will end in > instead of $. But everything else in this tutorial should look and work fine.

Lets navigate through folders.

Download and unzip the Garden of Forking Paths and place it into your Home Folder.

By default this will probably land in your Downloads folder. You should see a folder called ‘garden-of-forking-paths’. Move this folder into your ‘home’ folder that we will open using terminal.

On Mac, locate your ‘home’ folder by navigating to your desktop.
Click Go > Home. This where we will work from and store files in class, not on the Desktop. The reason for this is that when you clone and upload folders to github from terminal, this is the first folder that opens. It's easier to find your projects.

Enter this command to open finder in your home folder (Macs): open .

Folder Naming Practices

  • All lowercase filenames and folder names
  • No spaces
  • Use dashes or underscores. for example: my_file.txt or my-file.txt

Helpful Terms

Hope from one folder to the next with
cd [foldername]

Try to navigate through the garden-of-forking-paths folder by only using the command line tool.

Then navigate back to your home folder by typing cd .., and make a new directory by typing mkdir foldername. Name the folder whatever you would like. This will hold your site that you will create this week and any other files you create. Create an html file by typing touch index.html.

Make a new folder with
mkdir [foldername]

Make a new file with
touch [filename]

Remove file with

HTML Basics

Html Structure

The <body> tag defines the document's body.

The <body> element contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.

<html>
    <head>
        <title>Title of the document</title>
    </head>
    <body>
         <h1>This is a heading</h1>
         <p>This is a paragraph.</p>
    </body>               
</html>

Paragraphs

This is called a <p> tag, it is used to define a paragraph

<p>hello<p>

Browsers automatically add a single blank line before and after each <p> element.

The <br> tag inserts a single line break.

This is heading 1

This is heading 2

This is heading 3

This is heading 4

This is heading 5
This is heading 6
<h1>This is heading 1</h1>
 <h2>This is heading 2</h2>
 <h3>This is heading 3</h3>
 <h4>This is heading 4</h4>
 <h5>This is heading 5</h5>
 <h6>This is heading 6</h6>

Hyperlinks

this is an <a> tag 
<a href="#">this is an <a> tag</a>

The <a> tag defines a hyperlink, which is used to link from one page to another.

The most important attribute of the <a> element is the href attribute, which indicates the link's destination.

Div

The <div> tag defines a division or a section in an HTML document.

The <div> tag is used as a container for HTML elements - which is then styled with CSS or manipulated with JavaScript.

The <div>; tag is easily styled by using the class or id attribute.

Any sort of content can be put inside the <div> tag!

Span

The <span> tag is an inline container used to mark up a part of a text, or a part of a document.

The <span> tag is easily styled by CSS or manipulated with JavaScript using the class or id attribute.

The <span> tag is much like the <div> element, but <div> is a block-level element and <span> is an inline element.

Bold and Italic

The <strong> tag is used to define text with strong importance. The content inside is typically displayed in bold.

The <em> tag is used to define emphasized text. The content inside is typically displayed in italic.

Naming Your Page

The <title> tag defines the title of the document. The title must be text-only, and it is shown in the browser's title bar or in the page's tab.

The <title> tag is required in HTML documents!

The contents of a page title is very important for search engine optimization (SEO)! The page title is used by search engine algorithms to decide the order when listing pages in search results.

The <title> element:

  • defines a title in the browser toolbar
  • provides a title for the page when it is added to favorites
  • displays a title for the page in search-engine results

Lists

The <ul> tag defines an unordered (bulleted) list.

<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
 </ul>

The <ol> tag defines an ordered list. An ordered list can be numerical or alphabetical.

<ol>
    <li>Coffee</li>
    <li>Tea</li>
    <li>Milk</li>
 </ol>

Forms

The <form> tag is used to create an HTML form for user input.

The <input> tag specifies an input field where the user can enter data.

The <label> tag defines a label for several elements:

<form action="/action_page.php" method="get">
    <label for="fname">First name:</label>
    <input type="text" id="fname" name="fname"><br><br>
    <label for="lname">Last name:</label>
    <input type="text" id="lname" name="lname"><br><br>
    <input type="submit" value="Submit">
</form>




Images

The <img> tag is used to embed an image in an HTML page.

Images are not technically inserted into a web page; images are linked to web pages. The <img> tag creates a holding space for the referenced image.

The <img> tag has two required attributes:

  • src - Specifies the path to the image
  • alt - Specifies an alternate text for the image, if the image for some reason cannot be displayed
<img src="images/cat.png"> alt="a black and white cat jumping across grassy water">

Sizing your Image

By default, your image will appear as its original size. To resize the image, include width="value" inside the <img> tag and set the value to either a percentage like 100% or a fixed value like 100px.

Tip: To link an image to another document, simply nest the <img> tag inside an <a> tag.

Remember!

Putting ../ in a file path, means that you are going backwards, out of a current folder before you enter a new one. You will use this when you are referencing urls/file paths for <a> tags or <img> tags. If you see a 404 error or broken img icon, and you're confused, it might be because it is missing this notation to direct to the correct file in your folder structure.
