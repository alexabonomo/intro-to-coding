Self Publishing on the Web

10 Reasons to Self Publish on the Web

  1. Because it is fun.
  2. Because it is 100% your voice.
  3. Because it teaches you how to archive your work.
  4. Because it is more sustainable for the environment.
  5. Because you will always be in ownership of the code as long as you have it stored on your personal computer
  6. Because you will become a better writer and copy editor.
  7. Because you will take care of your own presence on the internet better.
  8. Because your data is less likely to become obsolete or unmaintained.
  9. Because you can change the layout at will.
  10. Because it will make you look cool and smart.

Folder Structure and Setting up a Project on Gitub

As a new media artist it is important to know how to self publish your work online. In order to do this, learning how to use git commands to publish to github is a very valuable skill.

On Mac:

Most versions of MacOS will already have Git installed and you can activate it in the terminal with git version. However, if you don't have Git installed for whatever reason, you can install the latest version of Git using one of several popular methods.

    Install git from an Installer

  1. Navigate to the latest macOS Git Installer and download the latest version.
  2. Open the command prompt "terminal" and type git version to verify Git was installed.

    3. Install git from Homebrew

  3. Install homebrew if you don't already have it, then: $ brew install git
  4. Once the command output has completed, you can verify the installation by typing: git version.

Github: a versioning tool where you can save and update your code.

Commit: a saved version of your code at a particular point.

Go to github and click on the top left corner icon > Your repositories

Repository = folder

Use git clone URL in the terminal.

Once you make your repository, there area a few settings.

  • Keep it public. You will need to in order to publish the site via Github Pages.
  • You may add a README file, its just a text file where you an write about your repository to folks.
  • Keep everything else the same. You can choose a license if you want. If you want to, MIT License is common, but its not really a big deal if you keep it none for small personal art sites.

First we are going to set up our project on github by creating a new repository.

  • Select "New"
  • Name your repository
  • Leave all the other settings and select "Create Repository"

Now that you have made a repository, click on it and click on the green button that says "Code".

Simply just copy the URL and open your terminal.

Type git clone url

the folder should show up in your home folder

cd foldername to navigate to the folder locally to start creating files.

When you are ready, you can drop your files inside your site folder into this folder that is connected to GitHub.

Once you place your files inside the folder, make sure you are inside the correct folder. You can look at the file path above where you type commands, or type ls to view all files to see that it looks like your folder. Then type git status.

Tt should appear like a list of red files. This means that you have "unstaged" files ready to be added the the "staging area". To add all unstaged files type git add ..

To see this change, type git status again. You should see all the files turn green. Which means that the files are now "staged" and ready to be commit.

Commit your change by writing git commit -m "your message".

Now you can take your commit and push it to github. Making it available on the internet! Type git push -u origin main or just git push.

You should see a bunch of messages cascade in your terminal and you will know that everything worked if you see a message with 100% in it. Or you can go to your GitHub repository webpage and see the files added.

Setting up Your Site to be Responsive

Once you start to make your websites, you want to be able to make the layouts look good on both mobile and desktop. For this, we use media queries.

Media queries is a piece of code that specifies what CSS styles are at a certain screen width. Usually we think of mobile devices as screens less than 600px.

In our CSS we can use:

@media only screen and (max-width: 600px) {
body {
        background-color: lightblue;
     }
}
